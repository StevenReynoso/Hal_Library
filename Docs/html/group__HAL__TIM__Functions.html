Initializes the specified timer to generate a 1 Hz overflow.

Sets up prescaler and ARR values to create a 1-second periodic timer overflow. Useful for software delay loops, polling, or LED blinking.

Parameters
timxTimer instance (e.g., TIM2, TIM3).
clk_hzTimer input clock frequency in Hz (APB1/APB2 dependent).
Note
Timer is started immediately. No interrupt or output configuration is done.

Initializes the specified timer to generate a 1 Hz overflow.

This sets up a basic timer using a fixed prescaler and a computed auto-reload value. When started, the timer will count up and overflow once every second.

Useful for:

  • Blinking an LED
  • Basic delay loops (without interrupts)
  • Timing functions where accuracy is not critical
Parameters
timxPointer to the TIMx peripheral (e.g., TIM2, TIM3).
clk_hzThe clock frequency driving the timer (in Hz).
Note
This function starts the timer immediately in upcounting mode.
Warning
Assumes a hardcoded prescaler of 1600.

Configures a specific channel of the timer for PWM output.

Sets PWM mode 1 on the given channel, sets duty cycle via CCRx, enables preload for smooth transitions, and enables channel output.

Parameters
timxTimer instance (e.g., TIM2).
channelChannel number (1–4).
dutyPWM duty cycle (0–ARR). Maps directly to CCRx.

Configures a specific channel of the timer for PWM output.

This configures a timer channel in PWM Mode 1, sets its duty cycle, and enables output. It only supports Channel 1 for now.

PWM Mode 1:

  • Output is HIGH while counter < duty
  • Output is LOW when counter ≥ duty
Parameters
timxPointer to the TIMx peripheral (e.g., TIM2).
channelPWM channel to configure (1–4).
dutyDuty cycle in ticks (0 to ARR value). Example: If ARR=100 and duty=25, output will be high for 25% of the time.
Note
Currently, only channel 1 is implemented. You can extend this function to handle CH2–CH4.

Initializes a timer for PWM output.

Configures prescaler and auto-reload register to define the PWM frequency. Use with tim_pwm_config_channel() to assign duty cycle and channel behavior.

Parameters
timxPointer to TIMx instance.
prescalerPrescaler value (PSC register) — pass raw value, not minus one.
arrAuto-reload value (ARR register) — pass raw value, not minus one.

Initializes a timer for PWM output.

Sets the prescaler and ARR (auto-reload) values so the timer can run in PWM mode. This does NOT configure any output channels. Use tim_pwm_config_channel() for that.

Useful for:

  • Controlling servo motors
  • Dimming LEDs
  • Generating analog-like voltages via filtering
Parameters
timxPointer to the TIMx peripheral (e.g., TIM2, TIM3).
prescalerTimer prescaler (raw value, not PSC - 1). Example: A value of 1600 divides the timer clock by 1600.
arrAuto-reload value — this sets the PWM period (raw). Example: An ARR of 100 creates a 0–99 count range.
Note
This function does NOT start the timer. Call tim_pwm_start() to begin output.

This enables the counter and applies all preload values immediately. It does NOT configure any output channels—only starts the base timer.
